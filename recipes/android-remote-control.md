# Codex × 安卓手机：扫码连接，远程操控

![Codex × 安卓手机：扫码连接，远程操控 场景图](../assets/recipes/android-remote-control.svg)

让安卓手机和桌面 Codex 任务保持协同，能查看线程、补充指令、确认低风险动作。

> 适合对象：想离开电脑后继续跟进桌面 Codex 任务的人。
> 最终产出：连接状态、配对检查表、远程操作记录、失败排查表

## 案例目标

这个案例不是让 Codex “讲讲怎么做”，而是让它交付一个能复查的工作结果。你要把输入、权限边界、验收标准提前说清楚，让 Codex 按“计划 -> 执行 -> 截图/文件 -> 验收”的顺序推进。

## 准备清单

- 安卓 ChatGPT App
- 桌面 Codex App
- 同一账号登录状态
- 稳定网络
- 二维码、配对或远程入口是否可见

## 推荐入口

| 项目 | 建议 |
| --- | --- |
| 推荐入口 | ChatGPT App / Desktop App / 同一账号 |
| 先做什么 | 让 Codex 只读检查输入和环境 |
| 再做什么 | 确认计划后执行生成、整理或验证 |
| 最后做什么 | 输出产物路径、截图、验证方法和风险说明 |

## 实操步骤

1. 确认手机和桌面使用同一账号。
2. 检查网络、版本和配对入口；如果界面没有入口，不强行虚构。
3. 在桌面端打开目标线程，确认任务可见。
4. 在手机端查看任务状态或追加指令。
5. 记录连接失败原因：账号、网络、防火墙、版本、权限。

## 可复制提示词

```text
请帮我检查安卓手机远程控制 Codex 的连接流程。要求：先列出账号、网络、二维码或配对入口、权限检查项；不要操作敏感页面；如果当前界面没有扫码入口，就给出失败排查表；最后记录手机端能否看到目标线程。
```

## 过程截图与配图

- 桌面端：目标线程或任务截图。
- 手机端：连接状态或失败提示。
- 排查表：账号、网络、版本、权限。

> 写教程或复盘时，建议把这些截图放在同名附件目录里。没有真实截图时，先保留“待补截图”占位，不要用与结果无关的装饰图冒充。

## 验收标准

- 手机能看到目标任务或明确失败原因。
- 补充指令能同步到桌面线程。
- 权限提示清楚。
- 敏感操作仍需人工确认。

## 常见风险

- 账号不一致会导致线程看不到。
- 网络或防火墙可能阻断配对。
- 手机端不要误确认发布、删除、支付等高风险动作。
