# Codex x README: 把文档一键变网页

![Codex x README: 把文档一键变网页 场景图](../assets/recipes/readme-to-web.svg)

## 案例目标

把 README 的核心信息整理成面向用户的静态首页，并把长教程移动到 docs/guide，案例移动到 recipes。

**最终产出**：一个能打开、能预览、能发布的静态首页。

## 适合谁

第一次想看到 Codex 真实产出的用户。

## 准备输入

- 现有 README.md
- 原有首页或空白 index.html
- 必须保留的国内站、购买、飞书教程链接

## 推荐提示词

```text
请把 README.md 的核心内容整理成一个静态首页。要求：保留所有原有国内站、备用入口、购买套餐、飞书教程链接；首页只放入口、学习路线和案例入口；长教程移动到 docs/guide，实战放到 recipes。完成后用本地预览检查。
```

## 执行流程

1. 先用 rg 搜索 README 中的 http 链接，列出必须保留清单。
2. 阅读现有 index.html 和 style.css，沿用已有视觉风格。
3. 规划首页信息架构：入口、套餐、学习路线、案例入口、FAQ。
4. 把长文本拆到 docs/guide 和 recipes，只在首页保留跳转。
5. 启动本地服务器，检查首页、docs、recipes、reference 能打开。

## Codex 应该交付什么

- 一份可复查的执行摘要。
- 关键文件或产物路径。
- 运行过的验证命令。
- 未完成事项和风险说明。

## 验收标准

- 首页 200 可访问。
- 原国内站、备用网址、购买和飞书链接仍能 grep 命中。
- 移动端没有横向滚动或按钮遮挡。

## 常见风险

- 删掉原有转化链接。
- 首页塞进过长文章，影响读者选择。
- 只看 HTML 没看实际浏览器截图。
