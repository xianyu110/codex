# Codex x 云服务器: 远程定位并修复 Bug

![Codex x 云服务器: 远程定位并修复 Bug 场景图](../assets/recipes/remote-bug-fix.svg)

## 案例目标

让 Codex 先读日志和状态，再提出最小修复，不泄露服务器凭据。

**最终产出**：排障记录、修复补丁、回滚方案。

## 适合谁

线上或测试服务器出现问题，需要远程排障的人。

## 准备输入

- 服务器访问方式
- 服务名
- 日志路径
- 复现 URL
- 安全边界

## 推荐提示词

```text
请帮我排查这台测试服务器的 500 错误。要求：不要打印密钥；先查看服务状态和最近日志；提出假设、实验和最小修复；修改前说明回滚方案。
```

## 执行流程

1. 确认连接方式和只读检查范围。
2. 查看服务状态、端口、最近日志。
3. 提出 2-3 个假设并逐一验证。
4. 做最小修复或生成补丁。
5. 重启/回滚前说明影响。

## Codex 应该交付什么

- 一份可复查的执行摘要。
- 关键文件或产物路径。
- 运行过的验证命令。
- 未完成事项和风险说明。

## 验收标准

- 错误可复现且修复后消失。
- 日志无新异常。
- 回滚方案明确。
- 没有输出密钥。

## 常见风险

- 在生产直接试错。
- 复制服务器密钥到聊天。
- 没有回滚方案。

## 复盘模板

```text
目标是否完成：
改动 / 产物：
验证命令：
验证结果：
保留或安全要求：
下一步：
```
