# Codex x Playwright MCP: 让 AI 操控浏览器

![Codex x Playwright MCP: 让 AI 操控浏览器 场景图](../assets/recipes/playwright-mcp.svg)

## 案例目标

让 Codex 使用浏览器实际打开页面，基于截图和 DOM 发现问题。

**最终产出**：浏览器截图、交互记录、问题报告。

## 适合谁

要测试网页、填表、截图、检查交互的人。

## 准备输入

- URL
- 测试目标
- 登录方式
- 期望截图或结果

## 推荐提示词

```text
请用 Playwright 打开本地页面 http://127.0.0.1:8765，检查首页、docs 入口和 recipes 入口是否能点击。要求：截图保存到 artifacts/，并汇报坏链接和控制台错误。
```

## 执行流程

1. 启动目标页面或确认 URL 可访问。
2. 打开浏览器并获取页面快照。
3. 点击关键入口并在每次导航后重新 snapshot。
4. 保存截图和控制台日志。
5. 汇总坏链接、布局问题和复现步骤。

## Codex 应该交付什么

- 一份可复查的执行摘要。
- 关键文件或产物路径。
- 运行过的验证命令。
- 未完成事项和风险说明。

## 验收标准

- 截图文件存在。
- 链接点击能到目标页。
- 控制台没有关键错误。
- 报告包含复现路径。

## 常见风险

- 只看 DOM 不看截图。
- 登录态不可用。
- 动态页面未等待加载完成。
