# Codex × Playwright MCP：让 AI 操控浏览器

![Codex × Playwright MCP：让 AI 操控浏览器 场景图](../assets/recipes/playwright-mcp.svg)

让 Codex 真正打开页面、点击入口、保存截图，并基于页面状态而不是想象来判断结果。

> 适合对象：需要测试网页、检查表单、验证导航、做截图留证的人。
> 最终产出：交互记录、截图、控制台错误、坏链接清单

## 案例目标

这个案例不是让 Codex “讲讲怎么做”，而是让它交付一个能复查的工作结果。你要把输入、权限边界、验收标准提前说清楚，让 Codex 按“计划 -> 执行 -> 截图/文件 -> 验收”的顺序推进。

## 准备清单

- 目标 URL 或本地服务地址
- 要验证的页面和按钮
- 是否需要登录以及登录方式
- 截图保存目录
- 不能点击的危险操作清单

## 推荐入口

| 项目 | 建议 |
| --- | --- |
| 推荐入口 | App / MCP / Browser |
| 先做什么 | 让 Codex 只读检查输入和环境 |
| 再做什么 | 确认计划后执行生成、整理或验证 |
| 最后做什么 | 输出产物路径、截图、验证方法和风险说明 |

## 实操步骤

1. 启动目标页面或确认线上 URL 可访问。
2. 让 Codex 打开页面并先做 snapshot，获得稳定元素引用。
3. 每次点击、跳转、弹窗后重新 snapshot。
4. 保存关键页面截图和控制台错误。
5. 输出复现路径、问题清单和修复建议。

## 可复制提示词

```text
请用 Playwright 打开 http://127.0.0.1:8765/recipes/，检查 16 个案例卡片是否都有图片、能否点击进入详情页。要求：每次跳转后重新 snapshot；截图保存到 output/playwright/；汇报坏链接、控制台错误和页面溢出问题。
```

## 过程截图与配图

- 首页截图：证明页面已打开。
- 点击截图：证明入口能跳转。
- 错误记录：控制台、404、网络失败。

> 写教程或复盘时，建议把这些截图放在同名附件目录里。没有真实截图时，先保留“待补截图”占位，不要用与结果无关的装饰图冒充。

## 验收标准

- 截图文件存在且能看到目标页面。
- 关键入口可点击到正确页面。
- 控制台没有关键错误。
- 报告包含复现步骤，而不是只写“正常”。

## 常见风险

- 需要登录时不要把密码贴进文档。
- 不要自动点击删除、发布、支付、发送按钮。
- 动态页面要等加载完成再下结论。
