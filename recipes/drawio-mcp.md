# Codex × Draw.io MCP：AI 自动绘制架构图

![Codex × Draw.io MCP：AI 自动绘制架构图 场景图](../assets/recipes/drawio-mcp.svg)

把系统说明、业务链路或接口调用关系转成可编辑架构图，而不是只生成一张不可修改的图片。

> 适合对象：需要给团队、客户或自己解释系统结构的人。
> 最终产出：.drawio 源文件、PNG 预览、节点关系说明

## 案例目标

这个案例不是让 Codex “讲讲怎么做”，而是让它交付一个能复查的工作结果。你要把输入、权限边界、验收标准提前说清楚，让 Codex 按“计划 -> 执行 -> 截图/文件 -> 验收”的顺序推进。

## 准备清单

- 系统背景说明
- 模块、服务、数据库、外部系统列表
- 调用方向或数据流
- 想要的图类型：架构图、流程图、泳道图或时序图
- 输出目录

## 推荐入口

| 项目 | 建议 |
| --- | --- |
| 推荐入口 | App / MCP / diagrams.net |
| 先做什么 | 让 Codex 只读检查输入和环境 |
| 再做什么 | 确认计划后执行生成、整理或验证 |
| 最后做什么 | 输出产物路径、截图、验证方法和风险说明 |

## 实操步骤

1. 让 Codex 先抽取实体、分组和连接关系。
2. 确认图类型和阅读方向，例如从左到右或从上到下。
3. 生成 .drawio XML，并导出 PNG 预览。
4. 检查节点是否重叠、连线是否反向、文字是否过长。
5. 把源文件、预览图和修改建议一起交付。

## 可复制提示词

```text
请根据这份系统说明生成一张 draw.io 架构图。要求：先列出节点、分组和连线关系；确认后输出 .drawio 源文件和 PNG 预览；节点分为入口、应用服务、数据存储、外部系统；连线要标注数据流，不能只给截图。
```

## 过程截图与配图

- 输入材料：系统说明或 README。
- 过程截图：节点关系草稿。
- 结果截图：PNG 预览和 draw.io 打开后的编辑界面。

> 写教程或复盘时，建议把这些截图放在同名附件目录里。没有真实截图时，先保留“待补截图”占位，不要用与结果无关的装饰图冒充。

## 验收标准

- .drawio 文件能被 diagrams.net 打开。
- 节点、分组、连线和方向符合事实。
- PNG 预览清晰，文字没有重叠。
- 团队可以继续手工编辑。

## 常见风险

- 不要让 Codex 根据猜测补不存在的服务。
- 复杂系统先画一张总览，再分图，不要一张图塞满。
- 内部系统名称如需脱敏，先约定替代名。
