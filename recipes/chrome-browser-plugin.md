# Codex × Chrome：让 AI 直接控制浏览器

![Codex × Chrome：让 AI 直接控制浏览器 场景图](../assets/recipes/chrome-browser-plugin.svg)

使用用户 Chrome 登录态完成明确、低风险的网页检查，例如后台筛选、页面核对、截图留证。

> 适合对象：需要在自己已登录的浏览器里检查页面，但不想手动重复点击的人。
> 最终产出：页面操作记录、截图、问题清单、安全边界说明

## 案例目标

这个案例不是让 Codex “讲讲怎么做”，而是让它交付一个能复查的工作结果。你要把输入、权限边界、验收标准提前说清楚，让 Codex 按“计划 -> 执行 -> 截图/文件 -> 验收”的顺序推进。

## 准备清单

- Chrome 控制能力或插件状态
- 目标网站和页面
- 允许点击的按钮
- 禁止点击的高风险按钮
- 截图和结果格式

## 推荐入口

| 项目 | 建议 |
| --- | --- |
| 推荐入口 | Chrome / 登录态 / 浏览器插件 |
| 先做什么 | 让 Codex 只读检查输入和环境 |
| 再做什么 | 确认计划后执行生成、整理或验证 |
| 最后做什么 | 输出产物路径、截图、验证方法和风险说明 |

## 实操步骤

1. 确认 Chrome 控制能力已启用。
2. 明确哪些页面可看、哪些按钮不能点。
3. 打开目标页并等待登录态加载。
4. 执行只读检查或低风险交互。
5. 截图并汇报页面状态、异常和未执行动作。

## 可复制提示词

```text
请使用 Chrome 打开这个后台页面，只查看列表、搜索和筛选功能。不要点击删除、发布、支付、发送、重置按钮。完成后截图并汇报筛选是否正常、页面是否报错、有哪些动作没有执行。
```

## 过程截图与配图

- 打开截图：确认是正确账号和页面。
- 操作截图：筛选前后状态。
- 结果记录：未点击的高风险按钮。

> 写教程或复盘时，建议把这些截图放在同名附件目录里。没有真实截图时，先保留“待补截图”占位，不要用与结果无关的装饰图冒充。

## 验收标准

- 截图显示正确页面。
- 没有触发高风险操作。
- 筛选、跳转或查看结果可复现。
- 报告说明哪些动作被刻意跳过。

## 常见风险

- 真实账号里不要做不可逆操作。
- 页面包含隐私数据时，不要把截图发到公开仓库。
- 验证码、支付、群发和删除动作保留人工确认。
