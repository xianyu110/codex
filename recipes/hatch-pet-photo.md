# Codex × Hatch Pet：用一张照片生成专属宠物

![Codex × Hatch Pet：用一张照片生成专属宠物 场景图](../assets/recipes/hatch-pet-photo.svg)

用一张参考照片或风格描述生成桌面宠物素材包，并记录安装、预览和回退方式。

> 适合对象：想把 Codex 桌面工作台做得更有个人风格，同时学习 Skill/插件安装流程的人。
> 最终产出：宠物素材包、安装路径、预览截图、回退说明

## 案例目标

这个案例不是让 Codex “讲讲怎么做”，而是让它交付一个能复查的工作结果。你要把输入、权限边界、验收标准提前说清楚，让 Codex 按“计划 -> 执行 -> 截图/文件 -> 验收”的顺序推进。

## 准备清单

- 参考照片或原创风格描述
- 宠物名称
- 动作状态：待机、思考、完成、提醒
- 当前 Codex App 或 Skill 页面
- 不覆盖已有素材的目录规则

## 推荐入口

| 项目 | 建议 |
| --- | --- |
| 推荐入口 | 桌面 App / Hatch Pet / Skill |
| 先做什么 | 让 Codex 只读检查输入和环境 |
| 再做什么 | 确认计划后执行生成、整理或验证 |
| 最后做什么 | 输出产物路径、截图、验证方法和风险说明 |

## 实操步骤

1. 确认是否已安装 Hatch Pet 相关技能或插件。
2. 检查参考照片是否可用，避免未经授权的人像或商业素材。
3. 让 Codex 生成宠物角色设定、动作清单和素材包。
4. 安装到指定目录后重载 App 或技能。
5. 预览桌面效果，并写清楚如何回退。

## 可复制提示词

```text
请帮我制作一个 Codex 桌面宠物。输入是一张参考照片和宠物名称。要求：先检查是否已安装 Hatch Pet 相关 skill；没有就说明安装步骤；生成宠物素材包、安装路径和预览截图；不要覆盖已有宠物素材；最后写回退方式。
```

## 过程截图与配图

- 输入证据：参考照片或风格描述。
- 生成证据：素材包目录和动作清单。
- 安装证据：桌面预览截图。

> 写教程或复盘时，建议把这些截图放在同名附件目录里。没有真实截图时，先保留“待补截图”占位，不要用与结果无关的装饰图冒充。

## 验收标准

- 宠物名称、素材包路径、安装路径明确。
- 安装后能在目标 App 或技能中看到。
- 不遮挡主要工作区。
- 回退步骤可执行。

## 常见风险

- 不要使用未经授权的人像或宠物照片做公开素材。
- 不要覆盖已有素材包。
- 如果当前 App 没有对应入口，就停在素材包和安装说明，不虚构按钮。
