# 推荐设置速查

这页给新手一个保守、可执行的 Codex 设置参考。原则是：先让 Codex 在受控范围内稳定完成任务，再逐步开放更多权限、工具和自动化。

## 默认推荐

| 设置 | 推荐值 | 原因 |
| --- | --- | --- |
| 默认语言 | 中文 | 适合本教程站读者 |
| 推理深度 | Medium | 兼顾速度和稳定性 |
| 复杂任务推理 | High / Extra High | 多文件、长链路、疑难 bug 再升 |
| 权限 | Workspace write / Auto | 日常改文件足够 |
| 网络 | 按需开启 | 查官方文档、安装依赖、部署时再用 |
| Plan Mode | 复杂任务开启 | 先确认范围再执行 |
| Thread 组织 | 一事一 Thread | 避免上下文混乱 |
| Git 策略 | 小步提交 | 方便 review 和回退 |

## 权限组合怎么选

| 任务 | 推荐权限 | 说明 |
| --- | --- | --- |
| 解释项目、做代码审查 | Read only | 不需要改文件 |
| 补教程、改网页、修小 bug | Workspace write | 只在当前项目内写入 |
| 安装依赖、查外部文档 | Workspace write + Network | 先确认命令和来源 |
| 跨目录整理素材 | Full Access | 先列路径，再执行 |
| 删除文件、清理缓存、重置 Git | 先人工确认 | 不要自动放行 |
| 自动化任务 | Workspace write + 明确日志 | 稳定流程再自动化 |

## 推理深度怎么选

| 任务类型 | 推荐 |
| --- | --- |
| 改错别字、补链接、简单 Markdown | Low / Medium |
| 单文件小改、普通前端样式 | Medium |
| 多文件教程改造、补测试、真实仓库修 bug | High |
| 架构迁移、疑难排障、长期目标 | Extra High |

不要用高推理掩盖坏提示词。先把目标、上下文、约束、完成标准写清楚。

## App 设置建议

| 设置区域 | 建议 |
| --- | --- |
| General | 保持默认即可；长任务可以防止系统休眠 |
| Keyboard Shortcuts | 记住命令面板、设置、终端、Diff、搜索 Thread |
| Agent configuration | 不懂高级配置时不要乱改 `config.toml` |
| Integrations & MCP | 只启用当前任务需要的 MCP |
| Browser | 先用于本地预览和公开页面，不处理敏感登录页 |
| Computer Use | 只给明确 GUI 任务，注意系统权限 |
| Personalization | 可以写“默认中文、简洁、先计划再改” |

## AGENTS.md 推荐规则

```md
## Rules

- 默认用中文解释。
- 保留原有国内站、备用网址、购买入口、飞书教程链接。
- 修改前先识别文件范围。
- 不要输出或写入任何 API Key、token、cookie、私钥。
- 不要新增统计、遥测或不必要网络请求。
- 行为变化要补测试；文档变化要给出验证方式。

## Commands

- 本地预览：python3 -m http.server 8765
- 搜索内容：rg "关键词"
- 检查 diff：git diff --check
```

## config.toml 适合放什么

`config.toml` 适合放 Codex 的本地设置，例如默认模型、权限策略、MCP 服务器、功能开关等。新手不要一开始就手改太多。

适合写：

- 默认 sandbox / approval 偏好。
- MCP server 配置。
- 项目级可写目录。
- 功能开关。

不适合写：

- API Key 明文。
- Cookie。
- 私钥。
- 一次性任务要求。

## Skills 设置建议

| 场景 | 建议 |
| --- | --- |
| 自己反复用的流程 | 放到用户级 skill |
| 仓库团队共享流程 | 放到 `.agents/skills` |
| 只本次要用 | 不做 skill，直接写 prompt |
| 需要外部工具 | Skill + MCP |
| 要给别人安装 | 考虑做成 plugin |

Skill 描述要写清楚触发条件。描述越清楚，Codex 越容易在正确任务里自动选择它。

## MCP 设置建议

| MCP 类型 | 风险 | 建议 |
| --- | --- | --- |
| 文档搜索 | 低 | 可作为查官方文档工具 |
| 浏览器自动化 | 中 | 限定网站和任务 |
| GitHub / issue / PR | 中 | 写操作前确认 |
| Figma / Notion / 飞书 | 中 | 注意私有数据范围 |
| 数据库 | 高 | 默认只读，限制查询范围 |
| 内部系统 | 高 | 先确认授权和日志 |

MCP 的关键不是“能不能连上”，而是“连上以后能做什么”。能写入、发送消息、改数据的 MCP 都要更谨慎。

## 自动化设置建议

自动化只适合已经跑通的稳定流程。

一条合格自动化至少写清楚：

- 触发频率。
- 输入来源。
- 去重状态。
- 输出位置。
- 日志路径。
- 失败汇报。
- 哪些步骤需要人工确认。

推荐输出格式：

```text
扫描数量：
新增数量：
成功数量：
失败数量：
日志路径：
下一步建议：
```

## 不建议默认开启

- 长期 Full Access。
- 对所有网站开放浏览器控制。
- 对数据库开放写权限。
- 自动执行删除、重置、转账、发布、群发消息。
- 把密钥写进配置文件或文档。

## 新手默认配置总结

第一周可以这样用：

1. 桌面 App 为主，CLI 为辅。
2. 权限默认 Workspace write。
3. 网络按需开启。
4. 大任务先 `/plan`。
5. 一条 Thread 只做一件事。
6. 每次完成后看 diff 和验证结果。
7. 反复出现的规则写入 `AGENTS.md`。
